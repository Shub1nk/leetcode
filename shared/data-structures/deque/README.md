# Deque

Двусторонняя очередь (double-ended queue) на **кольцевом буфере**. Базовый примитив — на его основе композицией собираются `Stack` и `Queue`.

## API

```ts
const dq = new Deque<number>();   // ёмкость по умолчанию 8

dq.pushFront(x);   // добавить в голову
dq.pushBack(x);    // добавить в хвост
dq.popFront();     // снять с головы → T | undefined
dq.popBack();      // снять с хвоста → T | undefined
dq.peekFront();    // посмотреть голову → T | undefined
dq.peekBack();     // посмотреть хвост → T | undefined
dq.size;           // количество элементов
dq.isEmpty();      // пустой ли
```

На пустом деке `pop*`/`peek*` возвращают `undefined` (идиома TS, как `Array.pop`) — исключений не бросаем.

## Реализация

Массив фиксированной длины + два поля состояния: `head` (индекс первого элемента) и `size` (количество). **Отдельного `tail` нет** — хвост вычисляется как `(head + size − 1) % capacity`.

- Логический индекс `i` → физический `(head + i) % capacity`.
- `push*`/`pop*` за концы двигают `head` и `size` по модулю; ветку про `Front` двигает `head`, ветку про `Back` — только `size`.
- При заполнении (`size === capacity`) — `resize`: новый массив ×2, элементы **разворачиваются** из кольца в линию (`head → 0`). Просто `slice` нельзя — данные могут быть разорваны через границу массива.

## Сложность

- `push*` / `pop*` / `peek*`: O(1) амортизированно (`resize` — редкий O(n)).
- Память: O(capacity).

## Заметки

- `pushFront`: `head = (head − 1 + capacity) % capacity` — слагаемое `+ capacity` обязательно, в JS `-1 % n === -1`.
- Порядок операций разный у концов: `pushBack` пишет **до** `size++`, `pushFront` двигает `head` **до** записи.
- `pop*` обнуляют освобождённую ячейку (`buffer[i] = undefined`) — гигиена памяти, чтобы не держать ссылку на снятый объект.
