# Queue

Очередь (FIFO — first in, first out) как **фасад над `Deque`**. Своей структуры данных не держит: композиция, внутри приватный `Deque`, наружу торчит только очередной API.

## API

```ts
const q = new Queue<number>();

q.enqueue(x);   // добавить в хвост
q.dequeue();    // снять с головы → T
q.peek();       // посмотреть голову → T
q.size;         // количество элементов
q.isEmpty();    // пустая ли
```

На пустой очереди `dequeue`/`peek` **бросают** `QueueUnderflowError` (сообщение экспортируется как `QUEUE_ERROR_EMPTY`). Это осознанно строже базового `Deque`, который на пустом отдаёт `undefined`: `dequeue`/`peek` всегда возвращают `T`, и вызывающему не нужно проверять `undefined`.

## Реализация

Композиция, а не наследование: `extends Deque` протащил бы наружу лишний API и сломал бы FIFO-инвариант. Поэтому `Deque` спрятан приватным полем, а очередь делегирует в **два конца** — кладём в хвост, забираем с головы:

- `enqueue` → `deque.pushBack`
- `dequeue` → `assertNotEmpty()` + `deque.popFront()!`
- `peek` → `assertNotEmpty()` + `deque.peekFront()!`
- `size` / `isEmpty` → проброс

`assertNotEmpty` даёт throw-контракт, `!` после него снимает `undefined` из типа (очередь гарантированно непуста).

## Сложность

- `enqueue` / `dequeue` / `peek`: O(1) амортизированно (наследует от `Deque`).
- Память: O(n).

## Заметки

- Именно кольцевой буфер под капотом даёт `dequeue` за O(1): снятие с головы — это сдвиг `head`, без сдвига всего массива (как было бы у `Array.shift`).
