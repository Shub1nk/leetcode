# 3. Longest Substring Without Repeating Characters

**Сложность:** Medium

**Ссылка:** https://leetcode.com/problems/longest-substring-without-repeating-characters/

## Условие

Дана строка `s`. Нужно найти длину самой длинной **подстроки** (непрерывного отрезка) без повторяющихся символов.

Важно: именно подстрока — символы идут подряд, а не подпоследовательность.

## Примеры

```
"abcabcbb"  → 3   // "abc"
"bbbbb"     → 1   // "b"
"pwwkew"    → 3   // "wke" (не "pwke" — она не подряд)
""          → 0
```

## Ограничения

- `0 <= s.length <= 5 * 10^4`
- `s` состоит из английских букв, цифр, символов и пробелов

## Решение

**Скользящее окно** с двумя указателями `left`/`right` и множеством `Set` символов, которые сейчас внутри окна `[left, right]`.

`right` ведёт цикл (один проход по строке), `left` подтягивается по необходимости — каждый указатель проходит строку максимум один раз.

Инвариант: окно `[left, right]` всегда содержит только уникальные символы.

Шаг итерации:
- берём `current = s[right]`;
- пока `current` уже есть в окне — **сжимаем слева**: удаляем `s[left]` из `Set`, `left++`. Это `while`, а не `if`: за один сдвиг дубликат может не уйти (`abcb` + `b` → надо вычистить до первого `b`);
- как только дубликат вычищен — добавляем `current` в `Set`;
- обновляем ответ: `result = max(result, right - left + 1)`.

Длину окна берём **из границ** (`right - left + 1`), а не копим отдельным счётчиком — так нечему рассинхрониться при сдвигах `left`.

## Сложность

- Время: **O(n)** — каждый символ добавляется в `Set` и удаляется из него не более раза
- Память: **O(min(n, m))** — в `Set` не больше символов, чем размер алфавита `m` или длина строки

## Заметки

- Ключевой момент — `while` на сжатии окна, а не `if`: левую границу двигаем, *пока* дубликат не исчезнет, а не на один шаг.
- Ответ через `right - left + 1` вместо инкрементального счётчика длины — устойчивее к ошибкам при скачках `left`.
- Альтернатива `Set` — `Map<символ, индекс>`: тогда `left` можно **прыжком** переставлять сразу за прошлую позицию дубликата, без пошагового `while`. Немного быстрее в худшем случае, но код менее наглядный — оставил `Set` ради читаемости.
