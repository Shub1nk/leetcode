# 2648. Generate Fibonacci Sequence

**Сложность:** Easy

**Ссылка:** https://leetcode.com/problems/generate-fibonacci-sequence/

## Условие

Необходимо реализовать **генераторную функцию** `generateFibonacciSequence`, которая возвращает объект-генератор. При каждом вызове метода `next()` генератор должен выдавать очередное число последовательности Фибоначчи.

Последовательность Фибоначчи задаётся рекуррентным соотношением:

```text
F(0) = 0
F(1) = 1
F(n) = F(n-1) + F(n-2),  n ≥ 2
```

Первые элементы: `0, 1, 1, 2, 3, 5, 8, 13, …`

## Примеры

**Пример 1**
```text
Ввод: callCount = 5
Вывод: [0, 1, 1, 2, 3]
Пояснение:
- gen.next().value → 0
- gen.next().value → 1
- gen.next().value → 1
- gen.next().value → 2
- gen.next().value → 3
```

**Пример 2**
```text
Ввод: callCount = 0
Вывод: []
Пояснение: next() ни разу не вызывается, поэтому ничего не получаем.
```

## Ограничения

- `0 ≤ callCount ≤ 50` — количество вызовов `next()`.
- Значения Фибоначчи помещаются в тип `number` (до `F(49) = 7778742049`).
- Внутри генератора не используется рекурсия — состояние хранится в двух переменных.

## Решение

Храним два соседних числа последовательности: `a` (текущее) и `b` (следующее). В бесконечном цикле выдаём `a` через `yield`, затем сдвигаем «окно»: следующее число `a + b` становится новым `b`, а прежнее `b` — новым `a`.

```typescript
export function* generateFibonacciSequence(): Generator<number, any, number> {
  let a = 0; // F(0)
  let b = 1; // F(1)
  while (true) {
    // выдаём текущее значение
    yield a;
    // вычисляем следующее и сдвигаем «окно»
    const next = a + b;
    a = b;
    b = next;
  }
}
```

### Логика решения

1. Инициализируем `a = 0` (F0) и `b = 1` (F1).
2. В бесконечном цикле `while (true)` выдаём `a` через `yield`.
3. Вычисляем следующее число `next = a + b`.
4. Сдвигаем окно: `a = b`, `b = next`.
5. Цикл повторяется, позволяя получить сколько угодно элементов.

Каждый `yield` возвращает текущий член, после чего состояние обновляется, так что следующий вызов `next()` отдаст следующий член. Поскольку цикл бесконечный, генератор обслуживает любое количество запросов в рамках ограничения `callCount ≤ 50`.

## Сложность

- **Время:** `O(k)` — `k` это количество вызовов `next()`, каждый вызов делает `O(1)` работы.
- **Память:** `O(1)` — хранится только два числа независимо от количества выданных элементов.

## Заметки

- Генератор бесконечный, поэтому в `for…of` его нужно ограничивать (например, через `break` по счётчику), иначе цикл не завершится.
- Итеративный подход с двумя переменными избегает переполнения стека, в отличие от наивной рекурсии.

## Связанное

- **Быстрое возведение в степень** — позволяет вычислять `F(n)` за `O(log n)` без генерации всех предыдущих чисел.
- **Итераторы и генераторы в JavaScript/TypeScript** — базовые конструкции, используемые в решении.
