**# 2187. Minimum Time to Complete Trips**

**Сложность:** Medium  

**Ссылка:** https://leetcode.com/problems/minimum-time-to-complete-trips/

## Условие
У вас есть массив `time`, где `time[i]` — количество единиц времени, которое требуется `i`‑й машине (автобусу) для выполнения одного полного рейса. Каждая машина может выполнять рейсы последовательно без перерыва.

Дано целое число `totalTrips` — минимальное количество рейсов, которое необходимо выполнить суммарно всеми машинами. Требуется найти **минимальное** количество времени, за которое суммарно будет выполнено хотя бы `totalTrips` рейсов.

## Примеры

**Пример 1**  
```text
Ввод: time = [1,2,3], totalTrips = 5
Вывод: 3
Пояснение: 
- За 3 единицы времени первая машина выполнит 3 рейса (⌊3/1⌋ = 3),
- вторая — 1 рейс (⌊3/2⌋ = 1),
- третья — 1 рейс (⌊3/3⌋ = 1).
Всего 5 рейсов, что удовлетворяет условию, и 3 — минимальное возможное время.
```

**Пример 2**  
```text
Ввод: time = [2], totalTrips = 1
Вывод: 2
Пояснение: Одна машина с временем одного рейса 2 требует ровно 2 единицы времени, чтобы выполнить один рейс.
```

## Ограничения
- `1 ≤ time.length ≤ 10⁵`
- `1 ≤ time[i] ≤ 10⁷`
- `1 ≤ totalTrips ≤ 10⁷`

## Решение
Для любого фиксированного момента времени `t` можно посчитать, сколько рейсов успеет выполнить каждая машина: `⌊t / time[i]⌋`. Сумма этих значений по всем машинам дает общее количество рейсов, выполненных к моменту `t`.

Функция `f(t) = Σ⌊t / time[i]⌋` монотонно не убывает, поэтому можно искать минимальное `t`, при котором `f(t) ≥ totalTrips`, с помощью бинарного поиска.

Алгоритм:
1. Установить левую границу `left` равной минимуму из `time` (минимальное время одного рейса).  
2. Установить правую границу `right` равной `left * totalTrips` — гарантированный верхний предел (одна самая быстрая машина делает все рейсы).  
3. Пока `left < right`:
   - Вычислить `mid = left + ⌊(right‑left)/2⌋`.
   - Посчитать `trips = Σ⌊mid / time[i]⌋`.
   - Если `trips ≥ totalTrips`, сдвинуть правую границу к `mid` (можно уменьшить время).  
   - Иначе сдвинуть левую границу к `mid + 1`.
4. После завершения цикла `left` (или `right`) будет минимальным требуемым временем.

Код (TypeScript) реализует описанный подход и использует вспомогательные функции `getMid` и `getSumTrips` для подсчётов.

## Сложность
- **Время:** `O(n · log(min(time)·totalTrips))`, где `n = time.length`.  
  На каждой итерации бинарного поиска просматривается весь массив `time`.
- **Память:** `O(1)` — только несколько переменных, без дополнительного массивного хранения.

## Заметки
- Брутфорс‑решение (постепенно увеличивая время) работает, но имеет линейную сложность по ответу и не проходит ограничения по времени для больших входов.  
- Бинарный поиск гарантирует логарифмическое количество проверок, что делает решение эффективным даже при максимальных ограничениях.  
- При подсчёте количества рейсов важно использовать целочисленное деление (`Math.floor`), чтобы избежать переполнения при больших значений `mid`.
