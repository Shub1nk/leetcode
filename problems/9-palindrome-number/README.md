# 9. Число-палиндром

**Сложность:** Easy

**Ссылка:** https://leetcode.com/problems/palindrome-number/

## Условие

Дано целое число `x`. Необходимо вернуть `true`, если `x` является палиндромом, и `false` в противном случае.

## Примеры

```
Вход: x = 121
Выход: true
Пояснение: 121 читается одинаково слева направо и справа налево.

Вход: x = -121
Выход: false
Пояснение: Слева направо читается как -121. Справа налево получается 121-. Поэтому не палиндром.

Вход: x = 10
Выход: false
Пояснение: Справа налево читается как 01. Поэтому не палиндром.
```

## Ограничения

- `-2^31 <= x <= 2^31 - 1`

## Решение

```typescript
export const isPalindrome = (x: number): boolean => {
  if (x < 0) return false;
  if (x < 10) return true;
  if (x % 10 === 0) return false;

  let reversed = 0;
  let original = x;

  while (original > 0) {
    reversed = reversed * 10 + (original % 10);
    original = Math.floor(original / 10);
  }

  return reversed === x;
};
```

### Логика решения

1. Отрицательные числа не могут быть палиндромами (из-за знака `-`)
2. Однозначные числа всегда палиндромы
3. Числа, оканчивающиеся на 0, не могут быть палиндромами (кроме самого 0)
4. Разворачиваем число и сравниваем с оригиналом

### Алгоритм

Разворот числа цифра за цифрой.

**Операции:**

- `x % 10` — получение последней цифры (остаток от деления на 10)
- `Math.floor(x / 10)` — отбрасывание последней цифры (целочисленное деление)

**Пример для x = 123:**

```
Итерация | original | original % 10 | reversed
---------|----------|---------------|----------
старт    | 123      | -             | 0
1        | 12       | 3             | 0 * 10 + 3 = 3
2        | 1        | 2             | 3 * 10 + 2 = 32
3        | 0        | 1             | 32 * 10 + 1 = 321
```

**Формула:**

```
reversed = Σ(d_i × 10^i)  где i считается от 0
```

Для `123`: `(1 × 10²) + (2 × 10¹) + (3 × 10⁰) = 100 + 20 + 3 = 321`

**Почему это работает:**

Каждая цифра "выталкивается" из original в reversed:
- `original % 10` забирает последнюю цифру
- `reversed * 10` сдвигает уже накопленные цифры влево
- Складываем — новая цифра становится последней

Это как очередь: читаем цифры справа налево и записываем их в новую переменную слева направо.

## Сложность

- **Время:** O(log₁₀(n)) — разворачиваем все цифры
- **Память:** O(1) — только несколько переменных

## Заметки

- Строковое преобразование не используется для лучшей производительности
