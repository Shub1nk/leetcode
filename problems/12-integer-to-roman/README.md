# 12. Integer to Roman

**Сложность:** Medium

**Ссылка:** https://leetcode.com/problems/integer-to-roman/

## Условие

Римские цифры представлены семью различными символами: `I`, `V`, `X`, `L`, `C`, `D` и `M`.

| Символ | Значение |
|--------|----------|
| I      | 1        |
| V      | 5        |
| X      | 10       |
| L      | 50       |
| C      | 100      |
| D      | 500      |
| M      | 1000     |

Римские цифры обычно записываются от большего к меньшему слева направо. Однако цифра для четырёх — это не `IIII`, а `IV`. Аналогично, для девяти используется вычитание: `IX` вместо `VIIII`. Вычитание используется в шести случаях:

- `I` перед `V` (5) и `X` (10) → `IV` (4) и `IX` (9)
- `X` перед `L` (50) и `C` (100) → `XL` (40) и `XC` (90)
- `C` перед `D` (500) и `M` (1000) → `CD` (400) и `CM` (900)

Дано целое число, преобразуйте его в римскую цифру.

## Примеры

**Пример 1:**
```
Вход: num = 3749
Выход: "MMMDCCXLIX"
Пояснение: 3000 = MMM, 700 = DCC, 40 = XL, 9 = IX
```

**Пример 2:**
```
Вход: num = 58
Выход: "LVIII"
Пояснение: 50 = L, 8 = VIII
```

**Пример 3:**
```
Вход: num = 1994
Выход: "MCMXCIV"
Пояснение: 1000 = M, 900 = CM, 90 = XC, 4 = IV
```

## Ограничения

- `1 <= num <= 3999`

## Решение

Жадный алгоритм: идём по таблице значений от большего к меньшему.

Таблица включает не только базовые символы, но и вычитающие комбинации (`CM`, `CD`, `XC`, `XL`, `IX`, `IV`), чтобы сразу получать корректные римские записи.

Для каждого элемента таблицы:
1. Вычисляем, сколько раз значение помещается в число: `quotient = floor(number / value)`
2. Если `quotient > 0`, добавляем символ `quotient` раз и уменьшаем число на остаток
3. Переходим к следующему (меньшему) значению

```typescript
const integerToRomanMap = [
  { key: "M", value: 1000 },
  { key: "CM", value: 900 },
  { key: "D", value: 500 },
  { key: "CD", value: 400 },
  { key: "C", value: 100 },
  { key: "XC", value: 90 },
  { key: "L", value: 50 },
  { key: "XL", value: 40 },
  { key: "X", value: 10 },
  { key: "IX", value: 9 },
  { key: "V", value: 5 },
  { key: "IV", value: 4 },
  { key: "I", value: 1 },
];

export const integerToRoman = (int: number): string => {
  let roman = "";
  let number = int;

  for (const romanItem of integerToRomanMap) {
    const { key, value } = romanItem;

    const quotient = Math.floor(number / value);
    const remainder = number % value;

    if (quotient > 0) {
      roman += key.repeat(quotient);
      number = remainder;
    } else {
      number = remainder;
    }
  }

  return roman;
};
```

### Пример для num = 1994

```
Значение | quotient | remainder | roman
---------|----------|-----------|--------
1000     | 1        | 994       | M
900      | 1        | 94        | MCM
500      | 0        | 94        | MCM
400      | 0        | 94        | MCM
100      | 0        | 94        | MCM
90       | 1        | 4         | MCMXC
50       | 0        | 4         | MCMXC
40       | 0        | 4         | MCMXC
10       | 0        | 4         | MCMXC
9        | 0        | 4         | MCMXC
5        | 0        | 4         | MCMXC
4        | 1        | 0         | MCMXCIV
1        | 0        | 0         | MCMXCIV
```

**Почему это работает:**

На каждом шаге берём максимально возможное значение из таблицы. Поскольку вычитающие комбинации уже включены в таблицу, жадный выбор всегда даёт каноническую римскую запись.

## Сложность

- **Время:** O(1) — фиксированная таблица из 13 элементов, `quotient` не превышает 3
- **Память:** O(1) — вспомогательные переменные; длина результата ограничена (до 15 символов для чисел ≤ 3999)
