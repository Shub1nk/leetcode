# 146. LRU Cache

**Сложность:** Medium

**Ссылка:** https://leetcode.com/problems/lru-cache/

## Условие

Спроектировать кэш фиксированной ёмкости по принципу **LRU** (Least Recently Used):

- `LRUCache(capacity)` — создать кэш с положительной ёмкостью.
- `get(key)` — вернуть значение по ключу или `-1`, если его нет.
- `put(key, value)` — добавить/обновить пару; при переполнении вытеснить самый давно используемый ключ.

Обе операции — в среднем за **O(1)**. «Использование» — это любое обращение, и `get`, и `put`.

## Примеры

```
["LRUCache","put","put","get","put","get","put","get","get","get"]
[[2],[1,1],[2,2],[1],[3,3],[2],[4,4],[1],[3],[4]]
→ [null,null,null,1,null,-1,null,-1,3,4]
```

`get(1)` освежает ключ 1, поэтому при следующем переполнении вытесняется 2, а не 1.

## Ограничения

- `1 <= capacity <= 3000`
- `0 <= key <= 10^4`
- `0 <= value <= 10^5`
- до `2 * 10^5` вызовов `get` + `put`

## Решение

Реализация на `Map<key, node>` + **двусвязный список** с фиктивными `head`/`tail` (sentinels):

- `Map` даёт O(1) доступ к узлу по ключу.
- Двусвязный список хранит порядок свежести: у `head` — самый свежий, `tail.prev` — кандидат на вытеснение.
- Двусвязный (а не односвязный) — чтобы вырезать узел из середины за O(1), зная только сам узел.
- Узел хранит **свой `key`** — он нужен, чтобы при вытеснении удалить запись из `Map` (`oldest.key`).
- Sentinels убирают краевые проверки: у любого реального узла соседи всегда не `null`.

`get`/`put` при попадании переносят узел в голову (`moveToHead`); `put` нового ключа при переполнении удаляет `tail.prev`.

> Разбор трёх подходов (наивный `sort` → порядок вставки `Map` → ручной список) и граф связей — в Obsidian.

## Сложность

- Время: O(1) — `get` и `put`
- Память: O(capacity)

## Заметки

- Порядок присваивания в `addToHead`: поле-«якорь» `head.next` перезаписывать **последним**, иначе теряется ссылка на бывший первый узел.
- На практике ручной список оказался быстрее варианта на порядке вставки `Map` (меньше работы с хэш-таблицей: `get` её не трогает, нет `delete`+`set`).
