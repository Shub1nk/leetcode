# 994. Rotting Oranges

**Сложность:** Medium

**Ссылка:** https://leetcode.com/problems/rotting-oranges/

## Условие

Дана сетка `grid` из клеток трёх видов: `0` — пусто, `1` — свежий апельсин, `2` — гнилой. Каждую минуту свежий апельсин, соседствующий с гнилым **по вертикали или горизонтали**, тоже становится гнилым.

Вернуть минимальное число минут, за которое сгниют все свежие. Если это невозможно (какой-то свежий недостижим) — вернуть `-1`.

## Примеры

```
[[2,1,1],
 [1,1,0],
 [0,1,1]]        → 4
```

```
[[2,1,1],
 [0,1,1],
 [1,0,1]]        → -1
```

Нижний-левый свежий апельсин заперт и не сгниёт никогда.

```
[[0,2]]          → 0
```

Свежих нет — ждать нечего.

## Ограничения

- `m == grid.length`, `n == grid[i].length`
- `1 <= m, n <= 10`
- `grid[i][j]` — это `0`, `1` или `2`

## Решение

Многоисточниковый **BFS** по сетке, послойно (одна «волна» = одна минута).

Отличие от обычного BFS — **не один старт, а много**: сетка может содержать несколько гнилых апельсинов в разных местах, и гниль расползается от всех одновременно.

**Фаза инициализации** (один проход по сетке):
- все гнилые (`2`) кладём в очередь — это «нулевой уровень», минута 0, все источники сразу;
- считаем `fresh` — сколько свежих на поле.

**Фаза BFS** — послойно, снимок размера уровня:
- на каждом уровне обрабатываем все клетки текущей волны, заражаем свежих соседей (`1 → 2`), уменьшаем `fresh`, кладём их в очередь как следующую волну;
- **минуту засчитываем только если уровень реально кого-то заразил** — это убирает off-by-one: и стартовый сид-уровень (ещё никто не сгнил), и последний уровень (некого заражать) не тратят минуту.

**Терминатор:** когда очередь пуста, фронт гниения дальше не растёт. `fresh === 0` → вернуть число минут; `fresh > 0` → остались недостижимые свежие → `-1`.

`visited` отдельный не нужен — **саму сетку используем как метку посещения**, меняя `1 → 2` в момент заражения. Уже гнилая клетка второй раз не заразится.

## Сложность

- Время: **O(m·n)** — каждую клетку кладём/достаём из очереди не более раза, соседей константа
- Память: **O(m·n)** — очередь в пике хранит до всех клеток одной волны

## Заметки

- `fresh` — один счётчик, закрывающий сразу три случая: `-1` (заперт), ранний `0` (свежих нет) и корректный останов BFS.
- «Минута ⇔ заразили кого-то на уровне» — центральный инвариант BFS: снимок `levelSize` до внутреннего цикла.
- Мутация входа (`1 → 2`) экономит память на `visited`, но разрушает `grid`.
