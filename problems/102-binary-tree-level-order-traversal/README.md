# 102. Binary Tree Level Order Traversal

**Сложность:** Medium

**Ссылка:** https://leetcode.com/problems/binary-tree-level-order-traversal/

## Условие

Дан корень `root` бинарного дерева. Нужно вернуть значения его узлов по уровням: массив массивов, где `i`-й подмассив — значения всех узлов на глубине `i`, слева направо.

## Примеры

```
root = [3,9,20,null,null,15,7]
→ [[3], [9,20], [15,7]]
```

```
    3
   / \
  9  20
     / \
    15  7
```

```
root = [1]
→ [[1]]
```

```
root = []
→ []
```

## Ограничения

- число узлов — в диапазоне `[0, 2000]`
- `-1000 <= Node.val <= 1000`

## Решение

Обход в ширину (**BFS**) по уровням через очередь.

Ключевой приём — **снимок размера уровня**. Перед обработкой уровня фиксируем текущий размер очереди `levelSize`: это ровно количество узлов на текущей глубине. Обрабатываем именно столько узлов, попутно докидывая их детей в хвост — эти дети образуют следующий уровень и не мешают, потому что мы отсчитали ровно `levelSize` итераций.

Инвариант: *в начале каждой итерации внешнего цикла очередь содержит ровно все узлы одного уровня и ничего лишнего.*

- **`levelSize` снимаем до внутреннего цикла.** Если завязаться на `queue.size` прямо в условии `for`, размер поплывёт по мере добавления детей — уровни слипнутся.
- В очередь кладём только непустых детей, поэтому внутри цикла узел всегда валиден.
- Дерево — частный случай графа (связный, ацикличный), поэтому `visited` не нужен: «назад» к родителю обхода нет. В графе с циклами он был бы обязателен.

## Сложность

- Время: **O(n)** — каждый узел ровно раз enqueue/dequeue
- Память: **O(n)** — на результат; сама очередь в пике — до `O(w)`, ширины дерева (`w ≤ n`)

## Заметки

- Очередь — `shared/Queue` (поверх `Deque`), даёт `O(1)` на `enqueue`/`dequeue`.
