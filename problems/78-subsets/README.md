# 78. Subsets

**Сложность:** Medium

**Ссылка:** https://leetcode.com/problems/subsets/

## Условие

Дан массив **уникальных** чисел `nums`. Вернуть все возможные подмножества (power set). Порядок подмножеств и элементов внутри — любой, дубликатов быть не должно.

## Примеры

```
nums = [1,2,3]
→ [[], [1], [2], [3], [1,2], [1,3], [2,3], [1,2,3]]   (в любом порядке)

nums = [0]
→ [[], [0]]
```

## Ограничения

- `1 <= nums.length <= 10`
- `-10 <= nums[i] <= 10`
- все элементы `nums` уникальны

## Решение

Бэктрекинг (DFS по дереву решений). На каждом элементе — бинарный выбор «взять / пропустить», поэтому дерево имеет глубину `n` и `2^n` листьев, каждый лист = готовое подмножество.

- Рекурсия `dfs(i, path)`: `i` — какой элемент решаем сейчас, `path` — накопленное подмножество.
- **Базовый случай** `i === nums.length` — все элементы решены, лист: кладём **копию** `[...path]` в результат.
- Иначе две ветки: `path.push(nums[i])` → `dfs(i+1)` → `path.pop()` (взять), затем `dfs(i+1)` (пропустить).

Схема *choose → explore → un-choose*: `pop` откатывает взятие перед веткой «пропустить». Рекурсия использует системный call stack — явный стек не нужен.

## Сложность

- Время: O(n · 2ⁿ) — `2ⁿ` подмножеств, копирование каждого до O(n).
- Память: O(n) вспомогательной (глубина рекурсии + `path`), не считая выходного массива.

## Заметки

- Копия `[...path]` на листе обязательна: `path` мутируется по ходу, без копии последующие `pop` испортят уже сохранённое.
- Базовый случай — первой строкой, до `nums[i]`: иначе на `i === length` прочитаешь `nums[length] === undefined`.
- Пустой вход `[]` даёт `[[]]` (одно пустое подмножество), а не `[]`.
