# 701. Insert into a Binary Search Tree

**Сложность:** Medium

**Ссылка:** https://leetcode.com/problems/insert-into-a-binary-search-tree/

## Условие

Дан корень `root` бинарного дерева поиска (**BST**) и значение `val`. Нужно вставить `val` в дерево так, чтобы свойство BST сохранилось, и вернуть корень обновлённого дерева.

Гарантируется, что нового значения в дереве ещё нет. Валидных ответов может быть несколько — принимается любое дерево, остающееся корректным BST.

## Примеры

```
root = [4,2,7,1,3], val = 5
→ [4,2,7,1,3,5]
```

```
root = [40,20,60,10,30,50,70], val = 25
→ [40,20,60,10,30,50,70,null,null,25]
```

`25 < 40` → влево к `20`; `25 > 20` → вправо к `30`; `25 < 30` → влево, а там пусто → `25` садится левым ребёнком `30`.

```
root = [4,2,7,1,3,null,null,null,null,null,null], val = 5
→ [4,2,7,1,3,5]
```

## Ограничения

- число узлов — в диапазоне `[0, 10^4]`
- `-10^8 <= Node.val <= 10^8`
- все значения узлов уникальны
- `-10^8 <= val <= 10^8`
- гарантируется, что `val` в исходном BST отсутствует

## Решение

Вставка идёт по тому же пути, что и поиск в BST: на каждом узле сравниваем `val` с `node.val` и уходим в одну сторону.

- **База:** дошли до `null` — это и есть место для нового листа, возвращаем `new TreeNode(val)`.
- **Шаг:** `val < node.val` → идём влево, иначе вправо.
- **Связывание:** результат рекурсии присваиваем обратно (`root[direction] = insertIntoBST(root[direction], val)`) и возвращаем текущий узел. За счёт присваивания `null`-ветка сама превращается в новый лист — отдельная проверка листа не нужна.

Направление выбирается один раз и используется как ключ (`root[direction]`), чтобы не дублировать ветки `left`/`right`.

Дубликатов по условию нет, поэтому равенство значений не обрабатываем.

## Сложность

- Время: **O(h)** — проходим один путь от корня до листа (`h` — высота: `O(log n)` для сбалансированного дерева, `O(n)` для вырожденного)
- Память: **O(h)** — глубина стека рекурсии

## Заметки

- LeetCode отдаёт готовый `TreeNode`, а массивы `[...]` — лишь сериализация ввода/вывода по уровням (BFS) с `null` для пропусков.
- Для локальных тестов рядом лежит `lib/`: `buildTree` (массив → дерево, BFS на очереди) и `treeToArray` (дерево → массив с обрезкой хвостовых `null`)
