# 104. Maximum Depth of Binary Tree

**Сложность:** Easy

**Ссылка:** https://leetcode.com/problems/maximum-depth-of-binary-tree/

## Условие

Дан корень `root` бинарного дерева. Нужно вернуть его максимальную глубину.

Максимальная глубина — количество узлов на самом длинном пути от корня до листа.

## Примеры

```
root = [3,9,20,null,null,15,7]
→ 3
```

Самый длинный путь: `3 → 20 → 15` (или `3 → 20 → 7`) — три узла.

```
root = [1,null,2]
→ 2
```

```
root = []
→ 0
```

## Ограничения

- число узлов — в диапазоне `[0, 10^4]`
- `-100 <= Node.val <= 100`

## Решение

Глубина дерева = глубина более «длинного» из поддеревьев плюс сам корень.

- **База:** дошли до `null` — узла нет, глубина `0`.
- **Шаг:** считаем глубину левого и правого поддеревьев рекурсивно, берём максимум и прибавляем `1` за текущий узел.

```
maxDepth(node):
  if node == null: return 0
  return max(maxDepth(node.left), maxDepth(node.right)) + 1
```

Это обход в глубину (DFS): рекурсия спускается до листьев, а на возврате «собирает» максимум высоты снизу вверх.

## Сложность

- Время: **O(n)** — каждый узел посещаем один раз
- Память: **O(h)** — глубина стека рекурсии (`h` — высота дерева: `O(log n)` для сбалансированного, `O(n)` для вырожденного в список)

## Заметки

- LeetCode отдаёт готовый `TreeNode`, а массивы `[...]` — сериализация по уровням (BFS) с `null` для пропусков.
- Альтернатива без рекурсии — BFS по уровням: считать число «слоёв» очереди. Даёт `O(n)` время и `O(w)` память (`w` — макс. ширина уровня), но для этой задачи рекурсивный DFS короче и нагляднее.
